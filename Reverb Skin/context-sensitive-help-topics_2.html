Creating Context-Sensitive Help Topics (DITA)
 
To create a context-sensitive help topic
Context-sensitive help topics provide users with immediate assistance without requiring them to leave the context in which they are working. They help answer questions like "What is this?" and "Why would I use this?", and they provide information for a particular object and its context. Many applications support context-sensitive help by allowing a user to click on a Help button on an application window. When the user clicks on the Help button, a context-sensitive help topic related to the application window the user is viewing, displays.
For example, within an application, you might see the following window.
If the user clicks on the Help button, then the context-sensitive help topic associated with the window displays.
ePublisher allows you to create context-sensitive help topics through the use of TopicAlias markers and topic IDs. In DITA, markers may be customized based on element such as <othermeta> . When creating a TopicAlias marker from <othermeta> , you specify the topic ID as the value of the content attribute. Applications use the topic ID to determine which topic to display when users click on the Help button on an application window.
In this DITA topic, we will create an <othermeta> element that has the name attribute defined as TopicAlias, and which has the content attribute set to IDH_CreatingContextSensitiveHelpTopicsDITA. The designer of the stationery has already assigned the appropriate context-sensitive behavior to the TopicAlias marker type in the ePublisher Design Project.
